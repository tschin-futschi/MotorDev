固件烧录流程 — 各空间与代码的对应关系

项目：MotorDev  |  范围：src/tabs/fwflashtab · src/services/fwflashservice · src/services/flashstrategy* · src/protocol/firmware_parser

一、整体数据流向

[磁盘文件 .bin/.hex]
        │
        │  parseFile() —— 静态解析，主线程
        ▼
[FirmwareInfo（栈/临时对象）]
   ├─ data: QByteArray         ← 合并后的连续二进制（烧录原料）
   ├─ segments / addr 信息     ← 仅显示用
   ├─ crc32 / effectiveBytes   ← 仅显示用
   └─ valid / errorMessage     ← UI 三页切换依据
        │
        │  FwFlashTab 拷贝 data 到成员 m_currentFirmwareData
        ▼
[FwFlashTab::m_currentFirmwareData (主线程, QByteArray)]
        │
        │  startFlash(icModel, firmware, totalBytes) —— 信号槽
        ▼
[FwFlashService 内部状态机 (主线程)]
   ├─ m_totalBytes
   ├─ m_cancelFlag (shared_ptr<atomic<bool>>)
   └─ runPreflightAndFlash()  ← 顺序触发 4 个停止回调
        │
        │  QThread::create lambda 拷贝 (firmware, cancelFlag, total, self)
        ▼
[Worker 线程栈]
   ├─ firmware (QByteArray 隐式共享，零拷贝)
   ├─ cancelFlag (shared_ptr 拷贝)
   └─ strategy (裸指针，registry 持有所有权)
        │
        │  strategy->flash(firmware, progress, *cancelFlag, &errorMsg)
        ▼
[具体 IC 策略实现：AW86006Strategy::flash() 等]
   └─ 真实串口/I2C 烧录算法（当前是 stub：1KB/100ms 推进）
        │
        │  progress(sent) → emit progressUpdated（跨线程 → QueuedConnection）
        │  invokeMethod 回主线程 setState(Completed/Failed/Cancelled)
        ▼
[UI 反馈：FwFlashControlPanel / FwFlashLogPanel / TopBar]

二、各"空间"与代码对应

1. 文件解析空间（主线程，瞬时）

  • 代码：src/protocol/firmware_parser.{h,cpp} → FirmwareParser::parseFile()
  • 输入：磁盘上的 .bin 或 Intel .hex
  • 输出：FirmwareInfo（栈对象）
  • 关键字段拆分：
    • data：烧录唯一原料。.bin 直读；.hex 按段合并、段间用 0xFF 填充成连续映像
    • segments / minAddress / maxAddress：只给 FwFileInfoPanel 显示，烧录算法不依赖
    • crc32 / effectiveBytes：显示+完整性校验用
    • valid / errorMessage：UI 状态切换依据（空 / 合法 / 错误三页）
  • 限制：MaxFileBytes = 1024 KB，超过即 valid=false

2. UI 暂存空间（FwFlashTab，主线程）

  • 代码：src/tabs/fwflashtab.{h,cpp}
  • 成员变量职责：
    • m_currentFilePath — 路径文本（行编辑显示）
    • m_currentFirmwareData — 解析后的 data 拷贝；点击"开始"时传给 Service
    • m_currentFirmwareTotal — 进度条总字节数
    • m_currentFileValid — 控制开始按钮使能（与 IC 选择共同决定）
  • 分工：FwFlashTab 不接触烧录算法，只做"路径 → 解析 → 暂存 → 启动 → 显示信号"

3. Service 状态空间（FwFlashService，主线程持有）

  • 代码：src/services/fwflashservice.{h,cpp}
  • 状态机：Idle → Preparing → StoppingScope → StoppingGenerator → StoppingCyclic → DisablingPmic → Flashing → Completed/Failed/Cancelled → Idle
  • 状态机切换只在主线程进行；Worker 通过 QMetaObject::invokeMethod 回到主线程才允许改 m_state
  • 关键内存对象：
    • m_cancelFlag: shared_ptr<atomic<bool>> — 跨线程共享的取消标志，UI 取消按钮在主线程置 true，worker 在循环里轮询
    • m_workerThread: QThread* — 烧录线程句柄，QThread::create 创建，finished 后 deleteLater 自动回收
    • 4 个 StopCallback — MainWindow 注入的"立即停止"回调（fire-and-forget，nullptr-safe）

4. 前置序列空间（主线程同步执行）

  • 代码：FwFlashService::runPreflightAndFlash() 上半段
  • 顺序：停采样 → 停发生器 → 停循环写入 → 关 PMIC
  • 协作约定：
    • 4 个步骤都是 invokeIfSet(cb)，回调本身用 QMetaObject::invokeMethod 把动作发到对应 Service 所在线程
    • 不等 ACK，不阻断；任一步抛异常仅记 warn
    • 完成后再检查一次 cancelFlag，被取消则直接进 Cancelled

5. Worker 线程空间（烧录算法运行环境）

  • 代码：FwFlashService::runPreflightAndFlash() 下半段（QThread::create lambda）
  • 线程栈上的拷贝/捕获：
    • firmware (QByteArray) — Qt 隐式共享，零深拷贝，worker 只读
    • cancelFlag (shared_ptr) — 引用计数 +1，保证 worker 比 service 析构晚也安全
    • total (qint64) — 进度回调用
    • self (QPointer<FwFlashService>) — 防御 service 提前析构
    • strategy (FlashStrategy*) — 裸指针，注册中心负责生命周期
  • 进度回调 progress(sent)：内部调 emit self->progressUpdated/stageMessage，跨线程 → QueuedConnection → 主线程更新 UI

6. 策略实现空间（具体 IC 烧录算法）

  • 代码：src/services/flashstrategies/{aw86006,aw86100,dw9786,dw9788}_strategy.{h,cpp}
  • 接口：FlashStrategy::flash(firmware, progress, cancelFlag, errorMessage)
  • 当前所有 4 款都是 stub：1KB/100ms 推进 progress、检查 cancelFlag，不发任何真实串口数据
  • 这就是你后续要填入的"真实烧录算法"位置；填入时：
    • 同步阻塞调用，直接在 worker 线程上跑
    • 通过 progress 回调上报字节数，不必逐字节
    • cancelFlag 翻 true 时下一安全点退出并 return false
    • 失败必须填 *errorMessage

7. 注册中心空间（策略目录）

  • 代码：src/services/flashstrategyregistry.{h,cpp}
  • 由 FwFlashTab 在构造时 make_unique 创建，Tab 持有所有权
  • FwFlashService 只持裸指针引用
  • IC 下拉框由 registry->all() 枚举产生；新增 IC 只需在 registerBuiltins() 加一行 add(...)

三、线程与所有权速查表

对象 创建/持有 运行线程 生命周期
FlashStrategyRegistryFwFlashTab (unique_ptr)主线程与 Tab 同
FlashStrategy*Registry (unique_ptr)主线程构造，Worker 调 flash与 Registry 同
FwFlashServiceFwFlashTab（QObject parent）主线程与 Tab 同
FirmwareInfo临时栈对象主线程单次解析作用域
m_currentFirmwareDataFwFlashTab 成员主线程直到选下一文件/清除
m_cancelFlagFwFlashService (shared_ptr)主线程置位 / Worker 读每次 startFlash 重建
m_workerThreadFwFlashService 创建独立 QThreadfinished → deleteLater
4 个 StopCallbackMainWindow 注入各自 Service 所在线程与 Service 同

四、你后续填策略代码时只要关注这一块

flashstrategies/<ic>_strategy.cpp 的 flash() 函数体，签名固定为：

bool flash(const QByteArray &firmware,
           std::function<void(qint64 sentBytes)> progress,
           const std::atomic<bool> &cancelFlag,
           QString *errorMessage);
只在 worker 线程运行，可直接同步调用 SerialManager::sendRawFrame （注意 SerialManager 在独立线程，必须走 QMetaObject::invokeMethod + QueuedConnection）， 其余 UI / 状态机 / 前置序列 / 进度展示框架已经全部接好，无需改动。
