# 示波器功能讨论摘要

Date: 2026-04-09
Status: discussion

## 1. 总体原则

- 示波器功能拆分为两部分：
  - 数据采集
  - UI 渲染
- 数据采集优先保证数据完整。
- UI 渲染优先保证快速、不失真。

## 2. 数据采集层

- 数据采集层包含：
  - 串口数据读取
  - 串口协议解析
  - 数据流帧拆包
- 数据流帧格式以 `protocol.md` 为准。
- 每个数据流帧都包含一次采样时刻的多通道数据。
- 各通道数据按协议定义的通道顺序排列。
- 采样运行期间，协议要求的通道都会出现在帧中。

## 3. 原始数据存储

- 每个通道各自维护一份原始环形缓冲。
- 最多支持 8 个通道。
- 原始环形缓冲容量由时间窗口和采样周期决定。
- 计算方式：
  - 原始窗口点数 = 时间窗口 / 采样周期
- 示例：
  - `4000ms / 100us = 40000` 点
- 每收到一帧流数据：
  - 每个启用通道写入 1 个新点
  - 缓冲满后，新点覆盖最旧点
- 环形缓冲初始值全部为 `0`。

## 4. UI 显示数据

- UI 不直接使用原始环形缓冲绘制。
- 每个通道额外维护一份 UI 显示用环形缓冲。
- UI 显示环形缓冲固定容量为 `3000`。
- UI 显示缓冲也按通道分别维护。

## 5. 原始数据到 UI 数据的压缩规则

- 原始数据进入 UI 前需要压缩处理。
- 压缩目标是降低渲染负担，并保持波形显示状态良好。
- 不采用 `min/max` 包络方式。
- 采用按桶压缩后取中位值的方法。
- 每个桶输出 1 个中位值。
- UI 使用这些中位值点连成折线显示。
- 原始数据写入时，同时推进 UI 数据构建过程。
- 原始数据与 UI 数据之间存在一个固定桶宽的取样过程。
- 不是每写入一个原始点就立即写入一个 UI 点。
- 只有当当前桶满后，才计算一次中位值并写入 UI 显示环形缓冲。
- 已经参与过压缩的数据不再重复处理。
- UI 显示数据按顺序写入各通道的 UI 环形缓冲。

## 6. 桶宽规则

- 桶宽由原始窗口点数和 UI 固定点数共同决定。
- 计算方式：
  - 桶宽 = 原始窗口点数 / 3000
- 桶宽按固定整数处理。
- 不使用浮动桶宽。
- 桶宽最小值为 `1`。

## 7. 配置变化时的行为

- 时间窗口由用户从预设选项中选择。
- 不允许用户自由输入时间窗口。
- 当前 UI 已经实现预设选择。
- 当时间窗口或采样周期发生变化时：
  - 原始环形缓冲重新初始化
  - UI 显示环形缓冲重新初始化
  - 当前桶状态重新初始化
  - 写指针、计数等状态变量重新初始化
- 重新初始化后，显示立即清零。
- 后续由新采样数据重新开始积累波形。

## 8. Start/Stop 与配置锁定

- 时间窗口切换只更新配置，不立即生效。
- 需要等到下一次 `Start` 才应用。
- `Start` 后整套采样配置锁定。
- `Stop` 后才能再次修改采样配置。

运行期间禁止修改的内容：

- 时间窗口
- 采样周期
- 通道启用状态
- 通道寄存器地址
- 通道描述文字

运行期间允许修改的内容：

- 缩放
- 平移
- 视图切换
- Y 轴自动/手动切换

## 9. Start 时的行为

- `Start` 时一次性读取当前全部配置。
- 生成本轮运行参数快照。
- 将该快照输出到 log。

## 10. 待继续讨论项

- Start 时 log 记录“用户配置值”还是“最终生效值”，或两者都记录
- UI 重绘触发策略
- 时间窗口预设列表的最终范围
- UI 环形缓冲在未填满时的展示细节
