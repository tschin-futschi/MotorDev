## Goal

重构 UI 框架：新增"配置"页面，将侧边栏从全局共享改为每个 Tab 独立拥有，删除原全局侧边栏的串口配置。

## Scope

### 1. 活动栏调整
- 最上方新增"配置"按钮，按钮顺序变为：配置 → 读写 → 烧录 → 示波（底部仍为设置）
- ActivityBar::Page 枚举新增 `ConfigPage = 0`，其余页面顺序后移

### 2. 侧边栏架构重构
- 删除 MainWindow 中的全局 Sidebar 实例和 sidebarToggleButton
- 每个 Tab 页面内部自带独立的侧边栏区域，结构为：

```
Tab (QWidget)
├── QHBoxLayout
│   ├── TabSidebar (QWidget)      — 每个 Tab 自己的侧边栏，可收缩
│   ├── SidebarToggleButton       — 收缩/展开按钮
│   └── TabContent (QWidget)      — 主内容区
```

- 现有 `Sidebar` 类重构为通用基类或可复用容器组件，提供收缩/展开动画能力
- 各 Tab 侧边栏内容不同，本阶段暂用占位文字标明用途

### 3. 新增"基础配置"页面（ConfigTab）
- 新建 `src/tabs/configtab.cpp` / `src/tabs/configtab.h`
- 侧边栏内容（占位）：马达厂家选择、配置文件选择
- 主内容区：COM 口配置区（端口、波特率、数据位、停止位、校验位下拉框 + 连接按钮），从原 Sidebar 迁移过来
- PMIC 配置、I2C 配置等区域先留空占位，后续增加

### 4. 其他 3 个 Tab 调整
- RegisterRwTab、FwFlashTab、OscilloscopTab 各自内嵌独立侧边栏
- 侧边栏内容本阶段为占位文字（标明"读写侧边栏"/"烧录侧边栏"/"示波侧边栏"）
- 主内容区保持当前占位 QLabel

### 5. MainWindow 简化
- 移除全局 Sidebar 和 toggleButton 相关代码
- middleLayout 变为：ActivityBar + ContentArea（QStackedWidget），不再包含 Sidebar
- QStackedWidget 包含 4 个 Tab（配置、读写、烧录、示波）

## Non-Goals

- PMIC / I2C 配置的具体参数和功能实现
- 各 Tab 侧边栏的具体功能内容（本阶段占位）
- 串口实际连接功能
- 任何通信逻辑

## Constraints

- **开始实现前必须先阅读 `docs/vision.md` 和 `design_spec.md`**
- Sidebar 重构后必须保持可复用性：各 Tab 能传入不同的内容 widget，共享收缩/展开动画逻辑
- 所有新增颜色、尺寸值必须添加到 `style_constants.h`
- 所有 UI 字符串使用 `tr()` 包裹
- 每个 Tab 类封装为独立文件，内部侧边栏布局自包含
- ConfigTab 的 COM 口配置区复用原 Sidebar 的下拉框逻辑（端口、波特率、数据位、停止位、校验位 + 连接按钮），但放在主内容区而非侧边栏
