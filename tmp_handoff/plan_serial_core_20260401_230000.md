## 详细设计

### 1. FrameParser

纯协议层，无 I/O 依赖。负责控制帧和数据流帧的编解码。

#### 1.1 数据结构

```cpp
// 解析后的控制帧
struct ControlFrame {
    uint8_t seq;        // 帧序号
    uint8_t cmd;        // 命令码
    QByteArray data;    // 数据段
};

// 解析后的数据流帧
struct StreamFrame {
    uint8_t channelMask;                // 通道掩码，每个 bit 对应一个通道
    QVector<int16_t> samples;           // 各通道采样值，按通道编号从低到高排列
};
```

#### 1.2 公共接口

```cpp
class FrameParser {
public:
    // 帧类型枚举，feedByte 解析成功时通过此枚举告知调用方帧类型
    enum class FrameType {
        None,       // 尚未解析出完整帧
        Control,    // 控制帧
        Stream      // 数据流帧
    };

    // 编码控制帧：输入 (seq, cmd, data) → 输出完整帧字节（含 AA 55 帧头 + CRC16）
    static QByteArray encodeControlFrame(uint8_t seq, uint8_t cmd, const QByteArray &data);

    // CRC16-MODBUS 计算（多项式 0x8005，初始值 0xFFFF，低位先行，结果大端输出）
    static uint16_t calculateCrc16(const QByteArray &data);

    // 逐字节喂入，返回解析出的帧类型
    // 解析出控制帧时通过 controlFrame() 获取结果
    // 解析出数据流帧时通过 streamFrame() 获取结果
    FrameType feedByte(uint8_t byte);

    // 获取最近一次解析成功的控制帧
    const ControlFrame &controlFrame() const;

    // 获取最近一次解析成功的数据流帧
    const StreamFrame &streamFrame() const;

    // 重置解析器状态（连接断开或出错时调用）
    void reset();

private:
    // 状态机状态
    enum class State {
        WaitHeader,     // 等待帧头首字节（0xAA 或 0xBB）

        // 控制帧状态
        WaitHeader2,    // 等待 0x55
        WaitSeq,        // 读帧序号
        WaitCmd,        // 读命令码
        WaitLen,        // 读数据长度
        WaitData,       // 读数据段
        WaitCrcHigh,    // 读 CRC 高字节
        WaitCrcLow,     // 读 CRC 低字节

        // 数据流帧状态
        WaitChannelMask,  // 读通道掩码
        WaitStreamLen,    // 读 LEN
        WaitStreamData,   // 读采样数据
        WaitXor           // 读 XOR 校验
    };

    State m_state = State::WaitHeader;

    // 控制帧解析临时变量
    uint8_t m_seq = 0;
    uint8_t m_cmd = 0;
    uint8_t m_len = 0;
    QByteArray m_data;
    uint8_t m_crcHigh = 0;
    ControlFrame m_controlFrame;

    // 数据流帧解析临时变量
    uint8_t m_channelMask = 0;
    uint8_t m_streamLen = 0;
    QByteArray m_streamData;
    StreamFrame m_streamFrame;
};
```

#### 1.3 关键实现要求

**控制帧编码/解码：**
- CRC16 校验范围：帧序号 + 命令码 + 数据长度 + 数据（不含帧头 AA 55）
- CRC16 结果**大端**存入帧（高字节在前），注意标准 MODBUS 库输出通常是小端，需要交换
- CRC 校验失败时丢弃该帧，回到 WaitHeader

**数据流帧解码：**
- 帧头 `0xBB`，后接通道掩码(1B) + LEN(1B) + 数据(nB) + XOR(1B)
- LEN = 通道掩码中 bit 为 1 的数量 × 2
- 各通道采样值为 int16，大端（高字节在前），按通道编号从低到高排列
- XOR 校验范围：通道掩码 + LEN + 数据（逐字节异或）
- XOR 校验失败时丢弃该帧，回到 WaitHeader
- 接收时应验证 LEN 与通道掩码的一致性，不一致则丢弃

**帧类型识别：**
- 状态机在 WaitHeader 状态根据首字节分流：`0xAA` → 控制帧路径，`0xBB` → 数据流帧路径
- 其他首字节 → 忽略，继续等待

### 2. SerialManager

通信管理层，持有 QSerialPort，对外提供信号槽接口。

#### 2.1 线程模型

使用 `QObject::moveToThread` 模式：
- SerialManager 本身是 QObject
- 创建 QThread，将 SerialManager move 到该线程
- QSerialPort 在 SerialManager 初始化时创建（保证线程亲和性）
- 外部通过信号槽调用（Qt 自动排队连接）

#### 2.2 公共接口

```cpp
class SerialManager : public QObject {
    Q_OBJECT

public:
    explicit SerialManager(QObject *parent = nullptr);
    ~SerialManager();

    // 枚举可用串口（静态方法，可在任意线程调用）
    static QStringList availablePorts();

public slots:
    // 初始化（在工作线程中创建 QSerialPort、定时器等）
    // 由 QThread::started 信号触发
    void init();

    // 打开串口（端口名 + 波特率）
    void openPort(const QString &portName, qint32 baudRate);

    // 关闭串口
    void closePort();

    // 发送控制帧（cmd + data），内部自动编码、分配帧序号、处理重发
    void sendCommand(uint8_t cmd, const QByteArray &data = {});

signals:
    // 连接状态变化
    void connected();
    void disconnected();

    // 收到解析后的控制帧响应
    void frameReceived(uint8_t cmd, uint8_t seq, const QByteArray &data);

    // 收到解析后的数据流帧（示波器采样数据）
    void streamDataReceived(uint8_t channelMask, const QVector<int16_t> &samples);

    // 错误（串口错误、指令超时等）
    void errorOccurred(const QString &message);

private slots:
    void onReadyRead();
    void onRetryTimeout();
    void onHeartbeatTimeout();

private:
    QSerialPort *m_serial = nullptr;
    FrameParser m_parser;

    // 帧序号管理
    uint8_t m_nextSeq = 0;

    // 重发机制
    QTimer *m_retryTimer = nullptr;
    QByteArray m_pendingFrame;      // 当前等待响应的已编码帧
    uint8_t m_pendingSeq = 0;       // 当前等待响应的帧序号
    uint8_t m_pendingCmd = 0;       // 当前等待响应的命令码
    int m_retryCount = 0;           // 已重发次数

    // 心跳
    QTimer *m_heartbeatTimer = nullptr;     // 1 秒发送间隔
    int m_missedHeartbeats = 0;             // 连续未响应次数

    static constexpr int MaxRetries = 2;             // 最多重发 2 次（共 3 次）
    static constexpr int RetryTimeoutMs = 100;       // 100ms 超时
    static constexpr int HeartbeatIntervalMs = 1000; // 1 秒心跳间隔
    static constexpr int MaxMissedHeartbeats = 3;    // 连续 3 次判定断线
};
```

#### 2.3 关键实现要求

**串口参数：** 8 数据位、1 停止位、无校验、无流控（protocol.md 第2节）

**发送与重发逻辑：**
1. `sendCommand()` 编码帧、写入串口、启动 100ms 重发定时器
2. 收到匹配响应（seq 匹配 或 cmd=0x01 错误响应）→ 停止定时器，发射 `frameReceived`
3. 100ms 超时 → 重发相同帧（帧序号不变），retryCount++
4. retryCount > MaxRetries → 停止定时器，发射 `errorOccurred("Command timeout")`
5. 同一时刻只允许一个待响应的命令（串行发送模型）

**心跳逻辑：**
1. 连接成功后启动 m_heartbeatTimer（1 秒间隔）
2. 每次触发发送心跳帧 `0x00`（心跳不走 sendCommand 的重发队列，直接编码写入）
3. 收到心跳响应（cmd=0x00）→ m_missedHeartbeats 归零
4. 心跳发送时 m_missedHeartbeats++，若 >= MaxMissedHeartbeats → 判定断线，发射 `disconnected`
5. 关闭串口时停止心跳定时器

**帧接收处理：**
1. `onReadyRead()` 读取所有可用字节，逐字节喂入 FrameParser
2. 根据 `feedByte` 返回的帧类型分别处理：
   - `FrameType::Control`：
     - 若 cmd=0x00（心跳响应）→ missedHeartbeats 归零，不发射 frameReceived
     - 若 cmd=0x01（错误响应）→ 停止重发定时器，发射 frameReceived 让上层处理错误码
     - 其他 → 检查 seq 是否匹配 pendingSeq，匹配则停止重发定时器，发射 frameReceived
   - `FrameType::Stream`：
     - 发射 `streamDataReceived(channelMask, samples)`

**连接/断开：**
- `openPort()` 成功 → 发射 `connected()`，启动心跳
- `closePort()` → 停止所有定时器，关闭串口，重置 FrameParser，发射 `disconnected()`
- 串口错误 → 发射 `errorOccurred()`
