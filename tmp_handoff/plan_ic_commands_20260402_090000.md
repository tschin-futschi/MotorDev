## Goal

实现 IC 框中 Scan 和 Connect 按钮的完整串口命令流程。Scan 发送 I2C 总线扫描命令（0x07），用响应中的地址列表填充 Slave ID ComboBox；Connect 发送设置电机 IC 地址命令（0x02），成功后更新 DeviceContext。可通过 SSCOM 手动回复响应帧进行测试。

## Scope

- ConfigTab 连接 `SerialManager::frameReceived` 信号，按 cmd 分发处理响应
- Scan 按钮发送 cmd=0x07, data=[0x02]（I2C2 总线）
- Connect 按钮发送 cmd=0x02, data=[slave_addr]
- 处理错误响应（cmd=0x01）
- 命令发送使用 `QMetaObject::invokeMethod` 跨线程调用

## Non-Goals

- 不修改 SerialManager / FrameParser 内部逻辑
- 不修改协议
- 不实现 PMIC 配置、寄存器读写等其他命令
- 不实现应答测试（0x05），后续需要时再加

                              .arg(addr, 2, 16, QLatin1Char('0')).toUpper();
    QByteArray data;
    data.append(static_cast<char>(addr));

        m_serialManager,

        Qt::QueuedConnection,
        Q_ARG(uint8_t, CmdSetMotorIcAddr),

### 4. onFrameReceived 实现

```cpp
void ConfigTab::onFrameReceived(uint8_t cmd, uint8_t seq, const QByteArray &data) {
    Q_UNUSED(seq);

    switch (cmd) {
    case CmdI2cBusScan:
        handleScanResponse(data);
        break;
    case CmdSetMotorIcAddr:
        handleSetAddrResponse();
        break;
    case CmdErrorResponse:
        handleErrorResponse(data);
        break;
    default:
        qDebug().noquote() << QStringLiteral("Unhandled frame: cmd=0x%1")
                                  .arg(cmd, 2, 16, QLatin1Char('0')).toUpper();
        break;
    }
}
```

### 5. handleScanResponse 实现

协议 0x07 响应数据格式：`[count][addr0][addr1]...[addrN-1]`

```cpp
void ConfigTab::handleScanResponse(const QByteArray &data) {
    if (data.isEmpty()) {
        qWarning() << "I2C scan response: empty data";
        return;
    }

    const uint8_t count = static_cast<uint8_t>(data.at(0));
    qDebug().noquote() << QStringLiteral("I2C scan found %1 devices").arg(count);

    const QSignalBlocker blocker(m_slaveIdCombo);
    m_slaveIdCombo->clear();

    for (int i = 1; i <= count && i < data.size(); ++i) {
        const uint8_t addr = static_cast<uint8_t>(data.at(i));
        const QString addrStr = QStringLiteral("0x%1")
                                    .arg(addr, 2, 16, QLatin1Char('0')).toUpper();
        m_slaveIdCombo->addItem(addrStr);
        qDebug().noquote() << QStringLiteral("  Device: %1").arg(addrStr);
    }

    if (count == 0) {
        qDebug() << "No I2C devices found";
    } else if (m_slaveIdCombo->count() > 0) {
        m_slaveIdCombo->setCurrentIndex(0);
    }
}
```

### 6. handleSetAddrResponse 实现

协议 0x02 成功响应：LEN=0（data 为空）

```cpp
void ConfigTab::handleSetAddrResponse() {
    const QString addrText = m_slaveIdCombo->currentText().trimmed();
    bool ok = false;
    const uint addr = addrText.toUInt(&ok, 16);
    if (ok && addr <= 0x7F) {
        m_deviceContext->setSlaveId(static_cast<uint8_t>(addr));
    }

    qDebug().noquote() << QStringLiteral("Motor IC address set to %1").arg(addrText);
}
```

### 7. handleErrorResponse 实现

协议 0x01 错误响应：data = [1 字节错误码]

```cpp
void ConfigTab::handleErrorResponse(const QByteArray &data) {
    uint8_t errorCode = 0;
    if (!data.isEmpty()) {
        errorCode = static_cast<uint8_t>(data.at(0));
    }

    QString errorName;
    switch (errorCode) {
    case 0x01:
        errorName = QStringLiteral("CRC check failed");
        break;
    case 0x02:
        errorName = QStringLiteral("Unknown command");
        break;
    case 0x03:
        errorName = QStringLiteral("Execution failed");
        break;
    default:
        errorName = QStringLiteral("Unknown error");
        break;
    }

    qWarning().noquote() << QStringLiteral("Error response: code=0x%1 (%2)")
                                .arg(errorCode, 2, 16, QLatin1Char('0')).toUpper()
                                .arg(errorName);
}
```

## 测试方法（SSCOM 模拟）

MotorDev 连 COM11，SSCOM 连 COM12。

### 测试 Scan

1. MotorDev 串口连接后，点击 IC 框的 Scan 按钮
2. MotorDev 发送帧（SSCOM 中可看到）：`AA 55 <seq> 07 01 02 <crc_hi> <crc_lo>`
3. 在 SSCOM 中用 HEX 发送模拟响应，例如发现 2 个设备（0x18 和 0x20）：
   - 需要构造帧：`AA 55 <same_seq> 07 03 02 18 20 <crc_hi> <crc_lo>`
   - seq 必须与 MotorDev 发送的 seq 相同
   - CRC 对 `<seq> 07 03 02 18 20` 计算
4. MotorDev 控制台应输出 "I2C scan found 2 devices"，Slave ID ComboBox 应填充 "0x18" 和 "0x20"

### 测试 Connect

1. 从 Slave ID ComboBox 选择一个地址（如 0x18）
2. 点击 Connect 按钮
3. MotorDev 发送帧：`AA 55 <seq> 02 01 18 <crc_hi> <crc_lo>`
4. 在 SSCOM 中发送成功响应：`AA 55 <same_seq> 02 00 <crc_hi> <crc_lo>`
5. MotorDev 控制台应输出 "Motor IC address set to 0x18"

### 测试错误响应

1. 对任何命令，SSCOM 发送错误响应：`AA 55 <seq> 01 01 03 <crc_hi> <crc_lo>`
2. MotorDev 控制台应输出 "Error response: code=0x03 (Execution failed)"
