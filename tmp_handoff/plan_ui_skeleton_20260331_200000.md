## Goal

从零搭建 MotorDev 项目的构建系统和完整 UI 骨架，使其能在本地编译运行，展示主窗口全部区域，点击活动栏可切换三个 Tab 页面。

## Scope

- CMakeLists.txt 基础构建配置（C++17, Qt6 模块）
- main.cpp 入口
- 主窗口框架 MainWindow，包含以下区域：
  - **TopBar**：Logo（QSvgWidget 22×22）、应用名"MotorDev"、串口信息标签（静态占位文字）、连接指示灯（红色圆点）、语言下拉框（占位，暂无实际切换逻辑）
  - **ActivityBar**：左侧固定 44px，3 个功能图标按钮（读写/烧录/示波）+ 底部设置按钮，点击切换 ContentArea
  - **Sidebar**：默认 150px，串口配置区（端口、波特率、数据位、停止位、校验位下拉框，静态占位数据）、底部"连接串口"按钮（无实际功能）、收缩/展开动画
  - **ContentArea**：QStackedWidget，3 个占位页面（各页面显示页面名称文字即可）
  - **StatusBar**：背景深绿，显示静态占位文字（串口参数 | 未连接 | RX: 0 | TX: 0 | 就绪）
- resources.qrc 资源文件 + motordev_logo.svg
- 所有颜色、尺寸提取为具名常量（头文件或命名空间），不硬编码在业务逻辑中
- 每个 UI 区域封装为独立 Widget 类

## Non-Goals

- 串口实际连接/通信功能
- 帧解析（FrameParser）
- Tab 1 寄存器表格的具体实现
- Tab 2 烧录功能
- Tab 3 示波器功能
- 多语言实际切换逻辑
- 配置持久化
- 数据日志

## Constraints

- **开始实现前必须先阅读 `docs/vision.md`，确保理解项目愿景和设计原则**
- **开始实现前必须先阅读 `design_spec.md`，所有颜色、尺寸以该文件为准**
- C++17 标准，Qt6 Widgets
- 所有颜色值、尺寸值必须定义在 `src/ui/style_constants.h` 中，以具名常量形式引用，禁止在 widget 代码中硬编码
- 每个 UI 区域（TopBar、ActivityBar、Sidebar、各 Tab）必须是独立的 Widget 类，有独立的 .h/.cpp 文件
- 布局使用 QHBoxLayout / QVBoxLayout / QSplitter，侧边栏收缩用 QPropertyAnimation
- 活动栏图标暂用文字标签代替（"读写"/"烧录"/"示波"/"设置"），后续替换为 SVG 图标
- 三个 Tab 占位页面只需一个 QLabel 居中显示页面名称
- 字符串使用 `tr()` 包裹，为后续多语言做准备
- 窗口默认 1280×800，最小 1024×600

## Notes

- Logo SVG 文件已在 `design_spec.md` 中定义了完整规格，工程师需据此创建 SVG 文件
- 侧边栏下拉框的选项为静态占位：端口（COM1/COM2/COM3）、波特率（9600/19200/38400/57600/115200/230400/460800/921600）、数据位（8）、停止位（1）、校验位（None）
- 活动栏图标文字用 9px 字体，后续阶段替换为实际图标
- 本阶段不需要 `serialmanager`、`frameparser`、`datalogger` 等模块
