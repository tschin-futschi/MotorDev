## 详细设计

### 1. 启用控制台窗口

在 CMakeLists.txt 中为 Windows 构建启用控制台子系统：

```cmake
if(WIN32)
    set_target_properties(MotorDev PROPERTIES WIN32_EXECUTABLE FALSE)
endif()
```

这样程序启动时会附带一个控制台窗口，qDebug/qWarning 输出会显示在控制台中。

### 2. 全局日志格式

在 `main.cpp` 中，`QApplication` 创建之前调用：

```cpp
qSetMessagePattern("[%{time yyyy-MM-dd hh:mm:ss.zzz}] [%{type}] %{message}");
```

输出效果示例：
```
[2026-04-02 00:30:15.123] [debug] Scan found 2 ports: COM3, COM4
[2026-04-02 00:30:18.456] [warning] Serial error: Failed to open serial port
```

### 3. 日志点定义

#### 3.1 ConfigTab (configtab.cpp)

| 操作 | 日志级别 | 日志内容示例 |
|------|---------|-------------|
| Scan 按钮点击 | debug | `Scan started` |
| Scan 结果 | debug | `Scan found 3 ports: COM3, COM4, COM5` 或 `Scan found 0 ports` |
| Connect 按钮点击（连接） | debug | `Connecting to COM3 at 115200...` |
| Connect 按钮点击（断开） | debug | `Disconnecting...` |
| Connect 无端口选中 | warning | `Connect failed: no port selected` |
| 连接成功回调 | debug | `Serial connected` |
| 断开回调 | debug | `Serial disconnected` |
| IC 型号变更 | debug | `IC type changed to DW9786` |
| Slave ID 变更 | debug | `Slave ID set to 0x18` |
| Slave ID 清空 | debug | `Slave ID cleared` |
| Slave ID 无效 | warning | `Invalid slave ID: 0xGG` |

#### 3.2 SerialManager (serialmanager.cpp)

| 操作 | 日志级别 | 日志内容示例 |
|------|---------|-------------|
| openPort 开始 | debug | `Opening port COM3 at 115200 (8N1)...` |
| openPort 成功 | debug | `Port COM3 opened successfully` |
| openPort 失败 | warning | `Failed to open COM3: <error string>` |
| closePort | debug | `Port closed` |
| 发送控制帧 | debug | `TX control frame: seq=0x01 cmd=0x00 len=0` |
| 接收控制帧 | debug | `RX control frame: seq=0x01 cmd=0x00 len=0` |
| 接收数据流帧 | debug | `RX stream frame: channels=0x03 samples=2` |
| 重发 | warning | `Retry #1 for seq=0x01 cmd=0x02` |
| 命令超时 | warning | `Command timeout: seq=0x01 cmd=0x02 after 3 attempts` |
| 心跳发送 | debug | `Heartbeat sent (missed: 1)` |
| 心跳响应 | debug | `Heartbeat response received` |
| 心跳断线 | warning | `Heartbeat lost: 3 missed, disconnecting` |
| 串口错误 | warning | `Serial port error: <error string>` |

## Acceptance

1. **编译通过** — `mingw32-make -C build_make_qt -j4` 成功
2. **控制台窗口** — 运行 MotorDev.exe 时弹出控制台窗口
3. **时间戳格式** — 所有日志输出带有 `[yyyy-MM-dd hh:mm:ss.zzz]` 时间戳前缀
4. **日志级别标签** — 每条日志带有 `[debug]` 或 `[warning]` 级别标签
5. **Scan 日志** — 点击 Scan 后控制台输出扫描结果（含端口数量和名称列表）
6. **Connect/Disconnect 日志** — 连接和断开操作在控制台有对应输出
7. **IC/SlaveID 日志** — 变更 IC 型号或 Slave ID 时控制台有对应输出
8. **SerialManager 日志** — openPort / closePort / 帧收发 / 重发 / 心跳 / 错误均有日志
9. **已有逻辑不变** — 日志仅为追加输出，不改变已有的业务逻辑和信号槽连接
10. **冻结文件未修改**
